# DDIC Self-LUT 图像批处理评估器设计

## 1. 文档信息
- 名称：DDIC Self-LUT 图像批处理评估器
- 日期：2026-03-09
- 目标：为 `V1` 全局对比度增强参考模型提供真实图片离线评估入口
- 范围：仅支持 `self-LUT` 模式，不模拟真实 `N -> N+1` delayed-LUT

## 2. 背景与目标

当前参考模型已经具备：
- `32/64 bin` histogram
- clip-limited CDF
- LUT 生成
- LUT IIR 状态封装

但仍缺少图像级入口，无法直接对真实图片观察增强效果。

本设计的目标是补齐一个轻量的离线评估工具，用于：

1. 读取一批真实图片
2. 对每张图片在亮度域上执行 `self-LUT` 对比度增强
3. 输出增强图、LUT 和统计信息
4. 为后续参数调试提供批量结果基础

## 3. 关键约束

- 当前测试阶段无法构造真实的 `N` 帧统计、`N+1` 帧应用闭环。
- 因此每张图只能使用自己的 histogram 生成自己的 LUT，再增强自己。
- 结果仅用于离线趋势验证，不等同于真实 DDIC 视频表现。
- 第一版不引入 GUI、视频、复杂指标、曲线绘图。

## 4. 方案选择

### 4.1 已确认选择
- 处理模式：目录批处理
- LUT 模式：`self-LUT`
- 颜色策略：只增强亮度，色度尽量保持不变
- 依赖策略：`Pillow + numpy`
- 输出策略：增强图 + `lut.csv` + `stats.json` + `summary.csv`

### 4.2 不纳入本次范围
- 真正的 delayed-LUT 序列仿真
- 视频文件直接输入
- 每张图自动生成 histogram/LUT 曲线图
- 大量客观画质指标

## 5. 总体架构

建议分成三层：

1. `reference_model.py`
- 保持算法核心
- 负责 histogram、clip、CDF、LUT 生成

2. `image_io.py`
- 负责图片读写
- 负责 RGB 到亮度域转换
- 负责亮度增强后重组输出图

3. `batch_runner.py`
- 负责扫描输入目录
- 调用图像处理流程
- 输出增强图和元数据
- 汇总整批结果

## 6. 数据流

对每张输入图片，处理流程为：

1. 读取图片为 RGB
2. 提取亮度通道 `Y`
3. 使用该图片自身的 `Y` 生成 histogram
4. 基于该 histogram 生成 LUT
5. 用该 LUT 映射该图片自身的 `Y`
6. 将增强后的亮度与原图色度重新合成
7. 保存输出图片与统计文件

由于是 `self-LUT` 模式，当前帧统计与当前帧增强绑定，不保留真实硬件时域语义。

## 7. 图像处理策略

### 7.1 颜色空间策略
第一版建议使用简单、稳定的亮度近似：

\[
Y = 0.299R + 0.587G + 0.114B
\]

增强只作用在 `Y` 上。

为降低色偏风险，建议输出阶段使用亮度增益恢复 RGB：

\[
gain = \frac{Y_{out}}{\max(Y_{in}, 1)}
\]
\[
R' = clip(gain \cdot R),\; G' = clip(gain \cdot G),\; B' = clip(gain \cdot B)
\]

该策略虽然不完全等价于 YCbCr 路径，但足够轻，且易于在离线工具中实现。

### 7.2 LUT 模式
第一版固定使用：
- 每张图独立计算 LUT
- 每张图独立应用 LUT
- 默认 `32 bin`
- 可通过命令行切换 `64 bin`

### 7.3 稳定性声明
由于 `self-LUT` 天然比 delayed-LUT 更“理想”，因此：
- 结果更适合看静态图像趋势
- 不适合判断视频闪烁和时域 pumping

## 8. CLI 设计

建议提供一个简单 CLI：

```bash
python -m ddic_ce.batch_runner INPUT_DIR OUTPUT_DIR [options]
```

首批参数：
- `--bins 32|64`
- `--clip-gain FLOAT`
- `--alpha-num INT`
- `--alpha-den INT`
- `--extensions png,jpg,jpeg,bmp`

说明：
- `alpha` 参数在 `self-LUT` 单张模式下影响较小，但保留是为了与参考模型配置统一
- 第一版不增加过多参数，避免工具变成调参面板

## 9. 输出目录结构

建议输出为：

```text
OUTPUT_DIR/
  enhanced/
    xxx.png
  lut/
    xxx.csv
  meta/
    xxx.json
  summary.csv
```

### 9.1 `enhanced/`
- 保存增强后的图片
- 文件名尽量与输入图一致

### 9.2 `lut/`
- 每张图保存一份 `256` 点 LUT
- 便于分析当前参数生成的 tone curve

### 9.3 `meta/`
- 每张图保存统计信息
- 建议包含：
  - 输入文件名
  - 宽高
  - `n_bins`
  - `clip_gain`
  - 输入亮度均值/最小值/最大值
  - 输出亮度均值/最小值/最大值
  - histogram 摘要

### 9.4 `summary.csv`
- 每张图一行
- 用于做横向参数对比和回归记录

## 10. 最小交付范围

本次实现只包含：

1. 目录级图片扫描
2. 图像读取与保存
3. 亮度域 `self-LUT` 增强
4. LUT 导出
5. 统计信息导出
6. 基础 CLI
7. 基础单元测试与集成测试

不包含：
- 视频序列模式
- 图表绘制
- 参数 sweep
- GUI

## 11. 测试策略

### 11.1 单元测试
- RGB 图转换亮度通道后尺寸和范围正确
- LUT 应用后亮度范围保持在 `[0, 255]`
- 灰度输入在 LUT 作用下输出合理
- 非图片文件被忽略

### 11.2 集成测试
- 构造临时输入目录
- 放入 1 到 2 张小图片
- 执行 batch runner
- 验证输出目录和文件是否生成完整
- 验证 `summary.csv` 行数与输入图片数一致

### 11.3 人工评估
建议准备：
- 暗场图
- 逆光图
- 人脸图
- 普通室内图

用固定参数跑批处理后，人工查看：
- 暗部层次是否提升
- 亮部是否发灰
- 色彩是否异常
- 过增强是否明显

## 12. 风险与取舍

### 12.1 风险
- `self-LUT` 结果可能高估真实硬件表现
- RGB 亮度增益重建可能带来轻微色彩变化
- 若 `clip_gain` 过高，静态图上也可能抬噪

### 12.2 当前取舍
- 优先要“能快速验证真实图效果”
- 暂时不追求完全复刻显示链色彩路径
- 暂时不追求视频时域一致性

## 13. 结论

本次推荐实现：
- `Pillow + numpy`
- 目录批处理
- `self-LUT` 单图增强
- 输出增强图、LUT 和统计文件

这是当前最小、最稳、最有评估价值的图像级验证路径。
